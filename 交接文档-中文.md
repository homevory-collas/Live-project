# UU9体育直播站 — 技术交接文档（中文）

> 交接对象：接手开发的技术团队。
> 本项目为前端 + Node 后端的体育/电竞直播门户。交付方将自行采购 CDN、版权、域名并继续开发。

---

## 一、项目概览

- 前端：纯静态（HTML/CSS/原生 JS，ES Modules），目录 `public/`。
- 后端：Node + Express，目录 `server/`。
- 存储：`server/db.js`，已装 `better-sqlite3` 用 SQLite，否则回退 JSON 文件。
- 多语言：中文 / 英文 / 越南语 / 马来语 / 印尼语（`public/js/i18n.js`）。
- 品牌名：UU9体育直播站。

> ⚠️ 重要：本项目由 AI 协助生成，**逻辑单元已单独测试通过**（密码哈希、邮箱验证流程、房间/广告增删改查、聊天分房广播、腾讯云签名），但**未在真实服务器上做过完整的端到端联调**（生成环境无法联网/装依赖）。请接手团队务必自行完整测试后再上线。

## 二、目录结构

```
public/                     前端（静态托管）
├─ index.html               首页
├─ live.html                直播间（HLS 播放 + 聊天 + 100 表情 + 游客 5 分钟限制）
├─ go-live.html             主播开播台（摄像头采集 + 贴广告 + 6 布景）
├─ admin.html               管理后台（建房 / 广告 / 地理封锁）
├─ auth.html                注册 / 登录 / 邮箱验证
├─ terms.html               用户协议（模板，需律师审核）
├─ contact.html             联系页
├─ assets/css/              样式（base.css 变量, app.css 界面）
└─ js/
   ├─ config.js             ★ 唯一需改的接入配置
   ├─ i18n.js               多语言字典（5 语）
   ├─ auth.js               前端登录态
   ├─ api/                  内部 API + 各服务商 adapter + mock
   ├─ ui/                   render / emojis(100) / ads
   └─ pages/                各页面逻辑

server/                     后端（Node）
├─ proxy.js                 主服务：API 代理 + 账号 + 房间 + 广告 + 静态托管
├─ chat.js                  聊天 WebSocket（分房广播）

.env.example                环境变量模板
```

## 三、后端接口清单

账号：
- `POST /api/auth/register` 注册（邮箱+密码，可选邮箱验证）
- `POST /api/auth/verify` 验证邮箱 6 位码
- `POST /api/auth/login` 登录
- `GET  /api/auth/me` 当前用户
- `POST /api/auth/logout` 登出

直播间：
- `GET  /api/live/rooms` 已发布房间列表
- `GET  /api/live/room?id=` 单房间（含地理封锁 451 / 腾讯云自动签名）
- `GET  /api/admin/rooms` 全部房间（需 ADMIN_TOKEN）
- `POST /api/admin/rooms` 新建/更新房间
- `DELETE /api/admin/rooms/:id` 删除

广告：
- `GET  /api/ads?slot=` 按位置取广告（按权重随机）
- `GET/POST/DELETE /api/admin/ads` 管理广告

数据代理：
- `GET /api/sports/scores`、`/api/sports/standings`、`/api/news`（转发到所购 API）

健康检查：`GET /health`

鉴权：管理与广告管理接口需 HTTP 头 `Authorization: Bearer <ADMIN_TOKEN>`。

## 四、环境变量（.env）

```
PORT                    服务端口（Render/Railway 自动注入）
ADMIN_TOKEN             管理后台令牌（务必改成强随机串）
REQUIRE_EMAIL_VERIFY    是否强制邮箱验证 true/false

# 体育数据 API（如 RapidAPI）
SPORTS_API_BASE / SPORTS_API_KEY / SPORTS_RAPID_HOST
NEWS_API_BASE / NEWS_API_KEY
LIVE_API_BASE / LIVE_API_KEY   （用外部直播间 API 时；留空则用本地存储）

# 邮件（发验证码）
MAIL_PROVIDER / MAIL_API_KEY / MAIL_FROM

# 腾讯云 CDN 防盗链
TENCENT_PLAY_DOMAIN / TENCENT_CDN_KEY / TENCENT_APP_NAME

# 聊天
CHAT_PORT
```

详见 `.env.example`。

## 五、本地运行

```bash
npm install
npm run proxy   # http://localhost:8080（同时托管前端）
npm run chat    # ws://localhost:8090
```
`public/js/config.js` 设 `USE_MOCK:false`，`CHAT_WS_URL` 指向聊天服务。

## 六、部署（Render 示例）

仓库含 `render.yaml`，Render → New → Blueprint → 选仓库 → Apply，
自动建 `uu9-web`（proxy，含前端）与 `uu9-chat`（chat）两服务。
之后在 `config.js` 把 `CHAT_WS_URL` 改为 `wss://<chat 域名>`。详见 `DEPLOY.md`。

⚠️ 免费档会休眠且文件存储会在重部署后丢失，仅用于测试；生产请用付费档 + 独立数据库（Postgres）+ 持久磁盘。

## 七、已完成 / 待开发（务必交接清楚）

✅ 已完成（前端 + 后端逻辑）：
- 首页 / 直播间 / 开播台 / 后台 / 注册登录五大页面
- HLS 播放（自恢复）、聊天 + 100 表情 + 飘屏、游客 5 分钟限制 + 横幅
- 注册/登录/邮箱验证逻辑、房间 CRUD、广告位、地理封锁（Web 层）
- SQLite/JSON 存储、5 语言、腾讯云签名函数

❌ / ⚠️ 待开发或需采购（接手方负责）：
- **赛事版权**（最关键、最贵，须合法授权）
- **视频源 → HLS 打包 → CDN**（腾讯云等，需自行采购配置）
- **地理封锁硬限制**：必须在 CDN/媒体层做（Web 层可被 VPN / 直取 .m3u8 绕过）
- **播放令牌硬限制**：游客 5 分钟为前端软限制，需后端签发时效令牌
- **邮件服务**：接 Resend/SES 才能真正发验证码
- **数据库**：生产换 Postgres + 持久化（现 JSON/SQLite 仅演示）
- **手机 APP**：当前为响应式网页，无独立 App
- **支付**：无（产品定位为免费 + 广告）
- **内容审核**：聊天与直播鉴黄需接入（腾讯云有鉴黄服务）
- **法律**：用户协议/隐私为模板，须律师按各市场（中/越/马/印尼）审核；OTT/直播牌照需合规
