# UU9体育直播站 · 运营与技术手册（中文）

> 面向：**运营者 / 管理员 / 技术维护人员**。
> 本手册逐部分讲解系统结构、部署、后台操作、接入第三方服务、常见故障排查。

---

# 第一章 · 系统总览

## 1.1 系统由三部分组成

| 部分 | 目录/文件 | 作用 | 运行方式 |
|---|---|---|---|
| 前端（界面） | `public/` | 用户看到的所有页面 | 静态托管 / 由后端一并托管 |
| 后端 API | `server/proxy.js` | 账号、房间、广告、地理封锁、隐藏密钥 | Node 运行 |
| 聊天服务 | `server/chat.js` | 直播间实时聊天（WebSocket） | Node 运行 |

辅助文件：`server/db.js`（存储）、`server/mail.js`（邮件）、`server/geo.js`（地理）、`server/tencent.js`（腾讯云签名）。

## 1.2 页面清单

| 页面 | 文件 | 用途 |
|---|---|---|
| 首页 | `index.html` | 直播列表、比分、积分、资讯、广告位 |
| 直播间 | `live.html` | HLS 播放器 + 聊天 + 100 表情 + 游客 5 分钟限制 |
| 开播台 | `go-live.html` | 主播用摄像头开播、贴广告、6 种布景 |
| 管理后台 | `admin.html` | 建房、管广告、设地理封锁 |
| 登录/注册 | `auth.html` | 邮箱注册登录、验证码 |
| 用户协议 | `terms.html` | 条款（需律师审核） |
| 联系我们 | `contact.html` | 联系方式 |

## 1.3 数据流

```
用户浏览器  ──HTTP──►  proxy.js (API + 托管前端)  ──►  存储(SQLite/JSON)
                                  │
                                  └──► 第三方API（体育/资讯，密钥在后端）
用户浏览器  ──WebSocket──►  chat.js（按房间广播聊天）
用户浏览器  ──HLS(.m3u8)──►  CDN（腾讯云等，视频在此分发）
```

# 第二章 · 本地运行（开发/测试）

## 2.1 仅看界面（无需后端）
`public/js/config.js` 设 `USE_MOCK: true`，然后：
```bash
npm run dev        # http://localhost:3000
```
用示例数据展示全部界面，适合演示设计。

## 2.2 完整运行（含后端）
```bash
cp .env.example .env     # 填写配置（见第五章）
npm install
npm run proxy            # 后端 + 网站  http://localhost:8080
npm run chat             # 聊天          ws://localhost:8090
```
再把 `config.js` 改为：
```js
USE_MOCK: false,
API_BASE: 'http://localhost:8080/api',
CHAT_WS_URL: 'ws://localhost:8090',
```

> 注意：摄像头开播页需 **HTTPS**，本地 localhost 例外可用；正式域名必须有 SSL。

# 第三章 · 管理后台操作（admin.html）

## 3.1 登录
1. 打开 `网址/admin.html`。
2. 输入 `ADMIN_TOKEN`（在 `.env` 设置的强随机串）。
3. 进入后台。令牌只存浏览器内存，刷新需重新登录。

## 3.2 新建/编辑直播间
逐字段说明：

| 字段 | 说明 | 示例 |
|---|---|---|
| 标题 | 直播间名称 | 英超 · 曼城 vs 利物浦 |
| 分类显示 | 显示用分类名 | 英超 / EPL |
| 分类键 | 用于筛选：football / basket / esport | football |
| 主播 | 主播/解说名称 | 体育频道 |
| 直播流 .m3u8 | **直接粘贴完整 HLS 链接** | https://.../index.m3u8 |
| 腾讯云流名 | 或只填流名，后端自动签名 | match001 |
| 封面图 URL | 封面图片地址 | https://.../cover.jpg |
| 允许观看的国家/地区 | 逗号分隔国家码，留空=不限制 | VN,MY,ID |
| 已发布 | 勾选后观众可见 | ✔ |

> 「直播流」与「腾讯云流名」二选一即可。若填了流名且配置了腾讯云，系统每次按需生成带时效签名的播放链接。

操作：填好 → 「保存」。列表中每行可「看 / 改 / 删」。

## 3.3 管理广告
| 字段 | 说明 |
|---|---|
| 位置 slot | home_top（首页顶）、home_side、live_below（播放器下）、live_preroll |
| 类型 | image（图片）或 html（自定义） |
| 图片 URL | 广告图地址 |
| 跳转链接 | 点击后打开的网址 |
| 自定义 HTML | type=html 时填写 |
| 权重 | 多条广告时按权重随机出现，数字越大越常出现 |
| 启用 | 关闭则不展示 |

保存后，对应页面的广告位 `<div data-ad-slot="...">` 会自动展示。

# 第四章 · 主播开播（go-live.html）

## 4.1 基本流程
1. 首页点红色「开播」或打开 `go-live.html`。
2. 「开启摄像头」→ 允许浏览器摄像头权限（需 HTTPS）。或「共享屏幕」。
3. 选布景：极简/竞技场/霓虹/球场/演播室/复古。
4. 贴广告元素（见 4.2）。
5. 填标题、分类 → 「开播」→ 计时开始 + 生成观众链接。
6. 「保存封面」截取当前画面。

## 4.2 广告元素
- 「加文字」：输入文字，选颜色与字号，鼠标**拖动**摆放。
- 「底部滚动条」：底部赞助商跑马灯。
- 「加 Logo/图」：上传图片并拖动。
- **双击**任意元素 = 删除。

## 4.3 重要限制
开播台只做**采集 + 本地预览**。要让观众真正看到，必须把摄像头流推到媒体服务器：
- WebRTC（WHIP）：填 `config.js` 的 `PUBLISH.WHIP_URL`；或
- 自建 RTMP→HLS 后由 CDN 分发。
仅本页不会对外播出。

# 第五章 · 环境变量（.env）逐项说明

| 变量 | 作用 | 备注 |
|---|---|---|
| `PORT` | 后端端口 | 默认 8080；云平台会自动注入 |
| `ADMIN_TOKEN` | 后台登录令牌 | **务必改成长随机串** |
| `SPORTS_API_BASE` | 体育数据 API 地址 | 见下 |
| `SPORTS_API_KEY` | 体育数据密钥 | RapidAPI 用 x-rapidapi-key |
| `SPORTS_RAPID_HOST` | RapidAPI 主机名 | 非 RapidAPI 留空 |
| `NEWS_API_BASE/KEY` | 资讯 API | 可选 |
| `LIVE_API_BASE/KEY` | 外部直播间 API | 留空则用本地后台管理的房间 |
| `REQUIRE_EMAIL_VERIFY` | 是否强制邮箱验证 | true/false |
| `MAIL_PROVIDER` | 邮件服务商 | 留空=验证码打印到日志；填 resend 启用真实发信 |
| `MAIL_API_KEY` | 邮件服务密钥 | |
| `MAIL_FROM` | 发件邮箱 | |
| `TENCENT_PLAY_DOMAIN` | 腾讯云播放域名 | 如 play.your-domain.com |
| `TENCENT_CDN_KEY` | 腾讯云防盗链 KEY | **保密，只放后端** |
| `TENCENT_APP_NAME` | 应用名 | 通常 live |
| `CHAT_PORT` | 聊天端口 | 默认 8090 |

> 安全铁律：`.env` 绝不提交到代码仓库；所有密钥只在后端。

# 第六章 · 接入腾讯云 CDN

## 6.1 你需要先在腾讯云做
1. 开通「云直播」，完成域名备案（中国大陆必须）。
2. 创建**推流域名**与**播放域名**。
3. 在「直播转码」开启多清晰度（建议；省带宽）。
4. 取得防盗链鉴权 **KEY**。

## 6.2 在 UU9 里配置
在 `.env` 填：
```
TENCENT_PLAY_DOMAIN=play.你的域名
TENCENT_CDN_KEY=你的鉴权KEY
TENCENT_APP_NAME=live
```
然后在后台建房时，「腾讯云流名」填你的流名（如 `match001`），系统会生成：
`https://play.你的域名/live/match001.m3u8?txSecret=...&txTime=...`

## 6.3 注意
- 本系统实现的是 **TypeA** 签名（txTime + txSecret）。若你在控制台选了其他鉴权类型，需对照腾讯云文档调整 `server/tencent.js`。
- 签名只能在后端做，KEY 不可暴露。
- 视频内容版权由你负责（赛事需授权）。

# 第七章 · 上线部署（Railway / Render）

详见 `DEPLOY.md`。要点：
1. 代码推到 GitHub。
2. Render：New → Blueprint（读 `render.yaml`）自动建 2 个服务（web + chat）。
3. 部署后把 `config.js` 的 `CHAT_WS_URL` 指向 chat 服务的 `wss://` 地址，重新部署。
4. 在平台环境变量里填 `.env` 的各项（尤其 `ADMIN_TOKEN`、腾讯云、邮件）。

> 免费套餐会休眠、数据库文件可能在重部署后丢失，正式运营请用付费套餐 + 持久数据库（Postgres）。

# 第八章 · 多语言

现有：中文 / EN / 越南语 / 马来语 / 印尼语。
新增语言：
1. `public/js/i18n.js` 的 `DICT` 复制一组（如 `th: {...}`），翻译所有键。
2. `public/js/config.js` 的 `LANGS` 加入语言码（如 `'th'`）。
界面无需改动。所有语言键必须**数量一致**，否则缺键会显示英文兜底。

# 第九章 · 内容安全与合规

- 直播间顶部已显示《文明观赛公约》。聊天服务可在 `chat.js` 的 `NOTE(moderation)` 处接入敏感词过滤。
- 涉黄/违规：可在腾讯云开「直播鉴黄」，或人工巡查。
- 地理封锁：后台设「允许观看的国家」。**注意这是网页层软封锁**，VPN 可绕过；硬封锁须在 CDN 层做（按版权合同要求）。
- 用户协议/隐私政策：`terms.html` 为模板，**正式上线须经律师按各市场法律审核**。

# 第十章 · 常见故障排查

| 现象 | 可能原因 | 处理 |
|---|---|---|
| 首页各栏空白 | `USE_MOCK:false` 但后端没起 / `API_BASE` 错 | 起后端；检查 config.js；或临时 `USE_MOCK:true` |
| 登录/注册无反应 | 后端未运行 / API_BASE 指向错 | 访问 `/health` 确认后端；修正地址 |
| 聊天不实时 | `CHAT_WS_URL` 没填或用了 ws:// 而非 wss:// | 线上必须 `wss://`（带 SSL） |
| 摄像头打不开 | 非 HTTPS / 浏览器拒绝权限 | 用 HTTPS 域名；检查权限 |
| 视频不播放 | .m3u8 链接无效 / 腾讯云签名类型不符 | 用真实域名测试链接；核对签名类型 |
| 后台进不去 | ADMIN_TOKEN 不一致 | 核对 `.env` 与输入值 |
| 部署后数据没了 | 免费套餐用文件存储，重部署丢失 | 用 Postgres 或持久卷 |

排错通用方法：浏览器按 **F12 → Console** 看报错；后端看运行日志。
